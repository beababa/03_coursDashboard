2.2 Grille

Comme les gomtries ne semblent pas se superposer, on utilise une grille dont chaque carreau fait 1 hac.

https://rcarto.github.io/carto_avec_r/chapitre3.html#les-grilles-r%C3%A9guli%C3%A8res

grid <- st_make_grid(bb, cellsize = 100)
plot(test2015$geometry)
plot(grid, add = TRUE)
inter <- function(data) { st_intersection(data, grid)}
linter <- lapply(ldata, inter)
sapply(linter, nrow)

La diffrence du nombre des carreaux montrent que les gomtries sont diffrentes.

Lobjet de rfrence est dsormais une liste.

Cela permet dappliquer rapidement les mmes traitements avec lapply et des fonctions spcifiques (cf fonction inter ci-dessus, permet dintersecter les gomtries avec la grille.

Inconvnient : manipulation des objets plus lourde ([[i]])

Avantage : viter les boucles

Exemple de fonction : calculer le % aire pour chaque carr de la grille Ceci permettra de donner une valeur continue chaque carreau. Par exemple, si on veut cartographier le % de mas sur chacune des 4 annes et comparer les cartes.

4.2.3 Nouvelle mthode : R + QGIS

En fait, on a besoin denregistrer le fichier de lanne et celui des 3 autres. On fera la diffrence sous Qgis. En effet, la diffrence ne ncessite pas de st_union et le traitement est moins long que sous R.

enregistrerAnneeUnique <- function(df, num) {
  zone <- deparse(substitute(df))
  print(zone)
  nomAn <- c("2015","2016","2017", "2018")
  X <- df [df$an == nomAn [num], ]
  Y <- df [df$an != nomAn [num],]
  # L'union peut se faire sous R quand mme non ?
  # Xunion <- st_union(X)
  # Yunion <- st_union(Y)
  # obligation d'enregistrer dans des gpkg sinon qgis ne les voit pas
  nomgpkg <- paste0("../dataS/chrysomelexport",zone,".gpkg")
  st_write(X, dsn = nomgpkg, layer = paste0(zone,"_", nomAn[num]), quiet = TRUE, delete_layer = TRUE)
  st_write(Y, dsn = nomgpkg, layer = paste0(zone,"_autres_", nomAn[num]), quiet = TRUE, delete_layer = TRUE)
}
